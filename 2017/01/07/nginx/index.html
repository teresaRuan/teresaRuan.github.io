简单学习nginx

Jan 7, 2017 | Hits

  之前在参加一个项目的时候接触了nginx服务器，觉得挺好用的，一直想学习下，总算是把这学习总结给写出来了。

  在开始之前，先给自己梳理几个概念：

  1. 正向代理 隐藏请求的客户端，由代理发送请求到已知唯一的服务器上
  2. 反向代理 是隐藏服务端，客户端通过Internet的请求由代理转发到内部网络上的服务器，然后代理把从服务器上得到的结果返回给客户端（客户端不知道具体是哪台服务器，可以用反向代理实现负载均衡）。看了知乎上的一个回答，感觉很清晰传送门
  3. 惊群现象 一个请求到来，多个睡眠进程被同时唤醒，但只有一个进程能获得链接，当再一个请求到来时会复现上述的现象，影响系统性能

–
  以下就主要常用的一些字段进行记录。

  参考nginx官网文档摘录如下：

常用

nginx -s stop 停止服务器

nginx -s quit 停止服务器，但不同于stop，quit是优雅的，如果进程还在服务中，那么不会关闭该进行直到进程完成
服务为止，此过程中不再接受新任务

nginx -s reload 重启服务器

nginx -s reopen 打开日志文件

  自行配置一个用于本地启动的nginx服务器，配置文件如下： 


worker_processes auto;

events {
    multi_accept on;
    worker_connections 10;
}

http{
    include mime.types;
    default_type  application/octet-stream;
    tcp_nodelay   on;
    send_file on;
    keepalice_timeout  65;

    gzip on;
    gzip_type text/css text/javascript;
    gzip_comp_level  4;

    upstram myServer {
        server  <ip>
        server  <ip> backup;
    }

    server {
        listen   80 default;
        server_name  my.test.server;
        charset utf-8;

        location / {
            root  <path>;
            index index.html index.htm
        }

        location <regx> {
            proxy_pass http://myServer;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }

    location ~ .*\.(gif|jpg|jpeg|png)$ {  
        expires 1d;  
    }   
    error_page   500 502 503 504  /50x.html;
    location = /50x.html {  
        root   <path>;
    }

    error_page  404              /404.html;
}


}

所遇问题

  windows下命令，在命令行输入命令的时候出现了问题，会报如下错误： 


oo@yaho MINGW64 /d/blog (master)
$ nginx
nginx: [alert] could not open error log file: CreateFile() "logs/error.log" failed (3: The system cannot find the path specified)
2017/01/07 20:56:53 [emerg] 7812#9092: CreateFile() "D:\blog/conf/nginx.conf" failed (3: The system cannot find the path specified)

原因： nginx没有启动

  故在命令窗口，输入 nginx.exe 启动nginx，打开任务管理器就会看到相应的进程，需要进行其他操作需要再开一个命令窗口

  之前看到有人说直接Ctrl+C 不能关闭服务器，我尝试了下，任务管理器内是已经没有这个进程了的。

  刚开始更改了了域名并没有做其他操作，一直无法访问服务。

原因： 没有配置hosts文件

  故更改域名后一定要更改hosts文件添加映射
